# ALTAIR DESIGN SYSTEM

Purpose:

Create a reusable experience layer for Altair OS before redesigning pages.

This is NOT a page redesign document.

This is NOT a feature document.

This is NOT a database document.

This document defines the reusable building blocks that every page will use.

---

## Build Order

### Phase 1 - Foundation

* GlassPanel
* StatusPill
* WorkspaceSection

### Phase 2 - Information

* MetricCard
* InsightCard
* PriorityCard

### Phase 3 - Actions

* HeroHeader
* ActionCard
* TimelineCard

### Phase 4 - Momentum

* PulseCard
* ProgressRing
* CelebrationBanner
* EmptyState

---

## Component Rules

Every component must answer one question.

HeroHeader

→ Where am I?

MetricCard

→ What is happening?

PriorityCard

→ What needs my attention?

InsightCard

→ What should I know?

ActionCard

→ What should I do?

PulseCard

→ How is my business feeling?

ProgressRing

→ How am I progressing?

TimelineCard

→ What happened?

CelebrationBanner

→ What went well?

EmptyState

→ What should I do first?

WorkspaceSection

→ How should this page be organized?

GlassPanel

→ How should information be contained?

StatusPill

→ What is the state of this item?

---

## Development Rules

Do NOT redesign pages.

Do NOT alter database schemas.

Do NOT alter routes.

Do NOT alter permissions.

Do NOT alter server actions.

Do NOT alter business logic.

Only build reusable presentational components that accept props.
