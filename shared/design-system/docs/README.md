# Altair Design System

Purpose:

This folder contains the reusable visual language for Altair OS V2.

Goal:

Transform Altair from a collection of business pages into a Living Operating System for trades businesses.

Rules:

* Do not alter business logic.
* Do not alter database architecture.
* Do not alter permissions.
* Do not alter routes.
* Do not alter server actions.
* Do not redesign pages directly.

Everything must be built as reusable components first.

Future Components:

* HeroHeader
* PriorityCard
* InsightCard
* MetricCard
* PulseCard
* ActionCard
* StatusPill
* CelebrationBanner
* EmptyState
* WorkspaceSection

Design Principles:

* Calm
* Intelligent
* Intentional
* Connected
* Helpful

Altair should answer:

"What should I do next?"

before showing data.
